Proyecto Final de Ciberseguridad
Análisis forense y endurecimiento de un servidor Linux (Debian)
Este repositorio contiene el proyecto final de ciberseguridad, desarrollado como parte de mi
formación práctica, centrado en el análisis forense y el endurecimiento de un servidor
Linux (Debian) ante un incidente de seguridad simulado.
El proyecto se realizó en un entorno de laboratorio local, utilizando máquinas virtuales
(VirtualBox), sin sistemas reales ni datos en producción. El objetivo principal fue aplicar, de
forma práctica, conceptos defensivos propios de un entorno Blue Team / SOC, más allá
de la explotación ofensiva.
Durante el desarrollo del proyecto se trabajó sobre un servidor Linux configurado de forma
deliberadamente vulnerable. A partir de este escenario, se analizaron posibles vectores
de riesgo, se revisaron servicios expuestos y se realizó un proceso de análisis de logs,
identificación de comportamientos anómalos y revisión de configuraciones
inseguras.
Como parte del trabajo, se llevaron a cabo tareas de escaneo y análisis de
vulnerabilidades, revisión de puertos y servicios activos, así como la evaluación de
debilidades comunes basadas en referencias como OWASP Top 10. Posteriormente, se
aplicaron medidas de endurecimiento (hardening) orientadas a reducir la superficie de
ataque del sistema, mejorar el control de accesos y reforzar la configuración general del
servidor.
El proyecto también pone énfasis en la documentación del proceso, la recogida de
evidencias y la redacción de conclusiones técnicas, simulando el tipo de trabajo que se
realiza en un entorno SOC, donde no solo es importante detectar un problema, sino
también entenderlo, documentarlo y proponer medidas de mitigación.
El resultado final del proyecto se encuentra recogido en el documento PDF incluido en este
repositorio, donde se detalla el contexto, el análisis realizado, las decisiones técnicas
adoptadas y las conclusiones obtenidas.
